Exhibit #3 — Nervous System

The "ESB" that isn't. One Function App + Service Bus proves the five integration reflexes. A trigger fires a function; the function reads or writes the bus. Every integration is a variation on that sentence.

The five reflexes

#patterntrigger → action
1Webhook ingress (two paths)WebhookDirect (HTTP) & EventGridIngress (Event Grid) → queue ingress
2Synchronous APIApiGet / ApiPost (HTTP) ↔ Cosmos
3Pub/sub fan-outpublish → topic events → ConsumerA + ConsumerB (both get a copy)
4Queue → adapter → externalQueueToTarget (SB queue) → POST to the API limb
5DB change → eventCosmosChangeFeed → queue db-events

Prove each reflex

Functions URL: (deploy with fn, then it appears here)

  1. Webhook (HTTP):
    curl -X POST <FN>/api/webhook -H 'content-type: application/json' -d '{"hello":"bus"}'
  2. Webhook (Event Grid): publish a CloudEvent to <EVENTGRID_ENDPOINT> — lands in the same ingress queue.
  3. API round-trip:
    curl -X POST <FN>/api/items -H 'content-type: application/json' -d '{"id":"a1","name":"demo"}'
curl <FN>/api/items/a1
  4. Fan-out (needs SB_TOPICS=1):
    curl -X POST <FN>/api/publish -d '{"evt":"hi"}'   # ConsumerA & ConsumerB both get it
  5. Adapter & change feed: a message on to-target is POSTed to the API limb; a new doc via ApiPost fires CosmosChangeFeed → db-events. Watch both in Application Insights.
