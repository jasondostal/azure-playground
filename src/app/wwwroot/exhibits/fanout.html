Exhibit #4 — Fan-out, two ways

One published event, two independent consumers, each getting its own copy — built two different ways in Azure. The AWS shape everyone knows: publish to an SNS topic, two SQS queues subscribed, a Lambda draining each.

Same job both ways: publish once → A and B each receive a copy independently. The difference is what carries it.
Service Bus topic — the SNS+SQS twin folded into one service. A subscription is a durable queue.
publish → topic 'fanout'
   ├─ subscription f-sub-a → Fn
   └─ subscription f-sub-b → Fn
Event Grid → Storage Queues — the same pattern from cheaper parts. Event Grid pushes; each Storage Queue is the durable buffer.
publish → Event Grid topic
   ├─ sub → queue fanout-a → Fn
   └─ sub → queue fanout-b → Fn

Service Bus folds the broadcast and the per-consumer buffer into one resource and brings the richer semantics (sessions, ordering, dead-letter). Event Grid + Storage Queues wire the same shape from two cheaper primitives. Both deliver one copy to A and one to B — that's the thing being shown.
