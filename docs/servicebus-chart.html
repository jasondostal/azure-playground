Service Bus for synchronous reads — the wrong tool

The inverse of "an API is fine": route the same SSN read through a message queue and watch it chug. Client-felt latency, LAN browser.
A queue is async decoupling infrastructure — not a way to answer a user who's waiting. ~7–18× slower than a plain API tier, it crosses a full second with naive polling, and on Basic (no sessions) it occasionally loses replies entirely. Slow and unreliable.
